Privacy Policy — Aristeem WebRTC/WS Logger

This policy describes how the Chrome extension “Aristeem WebRTC/WS Logger” (“Extension”, “we”, “our”) processes information.
Last updated: September 7, 2025 · Document version: 1.0

Who we are

Aristeem WebRTC/WS Logger is a developer tool used for diagnostics and quality monitoring within Aristeem projects. Contact: alextumarkin@gmail.com.

What this Extension does

The Extension observes technical events from the active page where it is allowed to run and produces JSON log files for debugging. It does not send data to any remote server. Logs are exported locally to the user’s Downloads folder either on a schedule or when the user clicks “Export”.

Data we process

The Extension collects technical telemetry from the page where it is enabled. Depending on the page activity, this may include:

  • WebRTC events: creation of RTCPeerConnection, session descriptions (SDP offer/answer), calls to setLocalDescription / setRemoteDescription, ICE candidates, connection state changes, and selective getStats() metrics (e.g., bytes sent/received, packets sent/received, jitter, round-trip time (RTT), available bitrate, etc.).
  • WebSocket events: connection open/close/error, WebSocket URL, sent messages, and received messages. For text messages we log up to ~1000 characters; binary messages are recorded as the marker [binary].
  • Page context: current page URL, origin, and document title (to help identify the source of events).

Note: WebRTC ICE candidates may contain your device’s IP addresses. WebSocket URLs or message contents could include identifiers if the visited site sends them. The Extension does not attempt to parse, enrich, or correlate such identifiers beyond basic logging.

Scope and limitations

  • The Extension runs only on sites you allow via its configuration (e.g., specific domains listed in the manifest).
  • It does not read arbitrary page content beyond the WebRTC/WebSocket data described above.
  • It does not access your browsing history, passwords, cookies, or form inputs.

Purpose of processing

Solely to support diagnostics, troubleshooting, and service-quality analysis for Aristeem-related development and support. We do not use the data for advertising, profiling, or selling to third parties.

Storage and retention

  • Events are buffered in the browser’s memory by the Extension’s background process.
  • On a timer (user-configurable, e.g., 1–10 minutes) or on user request, the buffer is exported as a JSON file to the local Downloads folder, then the in-memory buffer is cleared.
  • We do not transmit or back up logs to any server. You (or your organization) control the retention of the exported files.

Sharing and selling

We do not sell, rent, or trade collected data. We do not disclose it to third parties. No third-party analytics libraries are used.

Browser permissions used

  • downloads — to save JSON logs to your local Downloads folder.
  • alarms — to schedule periodic exports.
  • storage — to store user settings such as the export interval.

Security

Export happens locally on your device. Please treat the exported files as sensitive: store them securely, share only with trusted recipients, and delete them when no longer needed.

Your choices & controls

  • Use the popup to set the autosave interval (e.g., 1–10 minutes) and to trigger an immediate export.
  • Delete exported JSON files at any time from your Downloads folder.
  • Disable or remove the Extension via chrome://extensions if you no longer wish to use it.

Children’s privacy

This Extension is a developer tool and is not directed to children. It should be used by adults in a professional context.

Changes to this policy

We may update this policy if the Extension’s functionality changes (for example, if a future version introduces optional remote upload of logs). We will revise the “Last updated” date above. Material changes will be documented in the project repository and/or release notes.
